# 🛡️ Distributed Secure Message System (DSMS)

Разпределенный программный комплекс для криптографической защиты, сетевого логирования и оперативного администрирования текстовых данных. Проект разработан в рамках курсовой работы по дисциплине **«Высокоуровневые методы программирования»**.

Архитектура системы объединяет изолированное криптографическое ядро, локальное клиентское приложение, веб-сервис на базе Flask и интерактивную панель администратора для сквозного CRUD-контроля.

---

## 🚀 Ключевые особенности

* **Индивидуальное криптоядро (`Cypher.py`)**: Реализация алгоритма циклического сдвига (кастомный шифр Цезаря) с динамическим вычислением шага на основе `ID: 70225220` (сдвиг = 1). Расширенный алфавит `CIRCLE` (160 символов) обеспечивает полную обратимость кириллицы, латиницы и спецзнаков.
* **Кастомный Frameless UI (`App.py` / `Admin.py`)**: Полное отключение стандартных декораторов Windows (`overrideredirect`). Программно воссозданы перетаскивание окон, ресайз в 8-пиксельной зоне и уникальный дизайн в стиле "dark терминал" (`#1e1e1e` / `#00ff66`).
* **RESTful Web-Service (`Srv.py`)**: Легковесный сервер на **Flask**, принимающий POST-запросы, фиксирующий сетевые метаданные (IP, Time) и сериализующий логи в централизованную базу данных `messages.csv`.
* **Панель Администратора (`Admin.py`)**: Инструмент оперативно-диспетчерского управления с автоматическим фоновым обновлением данных (polling каждые 3 секунды), таблицей `Treeview` и поддержкой CRUD-операций.

---

## 📡 Спецификация Web-API (Srv.py)
| Маршрут | Метод | Описание | Тип ответа |
|---|---|---|---|
| /70225220 | GET | Отображение веб-интерфейса формы шифрования | HTML / CSS |
| /70225220 | POST | Прием текста, шифрование и запись в messages.csv | HTML (Статус успеха) |
| /get_all.json | GET | Чтение базы, дешифрование сообщений и отдача логов | JSON-массив объектов |
| /reset | GET | Деструктивный сброс — удаление файла messages.csv | Text / Terminal Log |
## 🔒 Безопасность и отказоустойчивость
> ⚠️ **Важно:** В модулях реализован безопасный контекстный менеджер try-except для операций файлового ввода-вывода. Это устраняет коллизии блокировки файла (PermissionError) при одновременном обращении Flask-сервера (запись) и панели администратора (чтение через автообновление).
